An ANSSI-compliant password — memorable, in two clicks.

SealKeeper is a self-hosted password distributor for enterprises. It helps your employees choose passwords that are at once compliant with ANSSI recommendations (B1/B2/B3) and easy to remember. The generator runs entirely in the user's browser via the WebCrypto API — the server never sees the secret, never stores it, has nothing to leak.

The problem we solve

Strong passwords your team will actually remember.

Every IT department faces the same dilemma. Either employees pick passwords they can remember, which usually fail an ANSSI audit. Or they pick something strong, write it on a sticky note, and effectively negate the strength. SealKeeper closes this gap.

I.

Weak passwords slip through

Left to their own devices, users gravitate to predictable patterns — birth years, names of children, keyboard sequences. They pass the minimum length check and fail every actual attack within hours.

II.

Strong passwords don't stick

A cryptographically random twenty-character string is unguessable and unmemorable. The post-it on the monitor or the unencrypted .txt file on the desktop are not a user failure — they are a predictable consequence.

III.

The middle ground

SealKeeper proposes passwords built from words, quotations or random alphanumerics — calibrated to ANSSI levels B1, B2, or B3 depending on the account's sensitivity. Same security floor, memorability tuned to the level. Generated in the browser, never seen by the server.

How it works

Four steps. Nothing crosses the server.

SealKeeper's distinguishing property is also its quietest. The password is generated inside the user's browser — entirely, with cryptographically secure randomness from the WebCrypto API. The server's role is reduced to gatekeeping: it verifies the email's domain is on the allowlist, decides which ANSSI policy applies, and emits a single-use email link. It never sees the password.

  1. Step 01

    Request

    The user opens the public page on the company intranet, types their work email, and submits. No other fields — the rest is decided by policy.

  2. Step 02

    Server gates

    The server checks the email's domain against the allowlist, determines the applicable ANSSI level (B1 by default, B2 or B3 if the address is in an admin-curated elevation list), and emits a single-use session token. Anti-enumeration: the public response is identical whether the address is allowed or not.

  3. Step 03

    Email link

    A signed email arrives in the user's inbox containing one link to the reveal page, parameterised with the session token. Default lifetime: fifteen minutes from issue, configurable per policy.

  4. Step 04

    Reveal & choose

    Clicking the link opens the reveal page. JavaScript fetches the applicable policy, generates N proposals in the browser using crypto.getRandomValues(), and displays them with their entropy. The user picks one, copies it to clipboard (auto-purged after thirty seconds), and the page closes itself when the session expires.

Three generators, three ANSSI levels

One floor of security. Three ways to remember.

SealKeeper ships with three generators, each calibrated to a level of the French ANSSI mot-de-passe reference. Same security floor, different memorability strategies depending on what the account protects.

G1

Citations transformed

For everyday accounts. A famous quote is picked at random from a corpus, transformed unpredictably (case, leet, separator), then sealed with three groups of three random digits. Memorable like a quote, scored at ANSSI B1.

123_EtreOuNeP@sEtre_456_789

~55 to 65 bits — ANSSI B1 ✓

G2

Word assembly (Diceware)

For accounts holding sensitive data. Six words are drawn independently from a five-thousand-word dictionary in the user's language, joined by a separator chosen at random, sealed with four random digits. Each word adds about thirteen bits of entropy.

cheval-table-pluie-roman-fenetre-jardin-4729

~90 bits — ANSSI B2 ✓

G3

Pure random, grouped

For administrators and privileged accounts. Twenty cryptographically random alphanumeric characters, presented in four groups of five for legibility — the same visual chunking as a software activation key. Maximum entropy at minimum length.

Aj7pK-2vBnR-qLcZ4-tFwM3

~119 bits — ANSSI B3 ✓

Generator and policy bound together. Each domain in the allowlist can configure up to three policies, one per ANSSI level. The administrator picks a generator per policy, uploads custom dictionaries or quotation corpora, and adjusts the parameters — with live entropy feedback in the console.
Zero-knowledge by design

The server has nothing to leak.

Most password tools have a moment where the plaintext exists somewhere reachable — a server process, a Redis key, a log line. SealKeeper doesn't. The password is generated inside the user's browser using the WebCrypto API and never leaves that browser through any HTTP request.

  • No password generation on the server. The server emits a policy descriptor (which generator, what parameters, applicable ANSSI level) and a single-use session token. The actual generation happens in the browser JavaScript, with crypto.getRandomValues().
  • No symmetric key on the server. There is nothing to encrypt or decrypt at the server tier — only policy, audit logs, and tokens. AES, GCM, RSA are absent from the server runtime.
  • Bundle integrity guaranteed. All JavaScript assets are served with Subresource Integrity hashes; the Content Security Policy forbids external script injection. The compiled bundle is reproducible-built and signed at release time.
  • No telemetry to third parties. No analytics scripts, no font CDN, no external API call. Everything is served by the same origin under your control.
  • Memory hygiene in the browser. Once the user copies their chosen password, the clipboard is auto-purged after thirty seconds via the Clipboard API. The reveal page closes itself when the session token expires.
For administrators

Everything you need to operate SealKeeper at scale.

The administrator console is the single place to configure, monitor, and audit SealKeeper across your domains. Authentication is gated by a strong password and a TOTP second factor, with optional WebAuthn for hardware key support.

★

Domain & policy management

Maintain an allowlist of authorised email domains. For each domain, configure up to three policies (one per ANSSI level) with the generator and parameters of your choice. Live entropy meter shows you exactly where you stand against ANSSI thresholds.

★

Elevation lists

Two curated lists per domain — addresses entitled to B2 (managers, functional admins) and addresses entitled to B3 (system administrators, security officers). All other addresses on the domain default to B1. Bulk import via CSV.

★

Editorial libraries

Upload custom word dictionaries and quotation corpora per language. The built-in defaults include EFF Diceware FR/EN and a public-domain quotation corpus. Use them, replace them, or extend them with your own editorial material.

★

Audit & observability

Every action — request, reveal, policy change, library upload — generates a signed audit event (HMAC chained per entry). Native export to syslog RFC 5424, JSON webhook, or push to Splunk, Sentinel, Elastic. Prometheus /metrics endpoint for Grafana dashboards.

★

Branding & localisation

Replace the SealKeeper logo and accent colour with your own. Customise the email subject, header, and signature. Bundled languages: FR, EN, ES, DE, IT — extensible via translation files.

★

SMTP & transport

Configure your SMTP relay, port, authentication, and TLS mode (STARTTLS, implicit). DKIM, DMARC, and SPF guidance bundled in the deployment documentation. Optional rate limiting per domain to prevent abuse.

Standards & references

Built on the right shoulders.

SealKeeper is engineered to align with widely recognised security and accessibility references. The aim is not paperwork compliance, but cumulative defence built on decisions other people debated for years.

  • ANSSI — *Recommandations relatives à l'authentification multifacteur et aux mots de passe*. Three levels of robustness (B1, B2, B3) directly mapped to the SealKeeper generators.
  • NIST SP 800-63B — *Digital Identity Guidelines*. Length over complexity, no forced rotation, no composition rules unless warranted.
  • OWASP ASVS 4.0 — Authentication Verification Standard, level 2 baseline for the console.
  • RGAA 4.1 — French accessibility reference; SealKeeper targets level AA, including keyboard navigation, screen reader compatibility, and audio playback of generated passwords for visually impaired users.
  • RGPD (GDPR) — Configurable retention of audit logs by the administrator, DPO export, data minimisation by design (no password stored, no plaintext logged).
  • RFC 6238 (TOTP) — Time-based one-time passwords for admin console second factor.
  • W3C WebCrypto API — Standardised cryptographic primitives in the browser, used for all in-page generation.
Quickstart

Evaluate SealKeeper locally in five minutes.

SealKeeper is distributed as a single Go binary and as an OCI image. The command below brings up an evaluation instance with an embedded SMTP capture (no real emails sent), a SQLite store, and the default EFF Diceware libraries pre-installed. It is not a production configuration — see the deployment guide for that.

I. Start an evaluation instance 
# evaluation mode: SMTP capture, SQLite, default libraries
docker run --rm -p 8443:8443 \
  -e SK_MODE="eval" \
  -e SK_BASE_URL="https://localhost:8443" \
  ghcr.io/sealkeeper/sealkeeper:latest
II. Open the admin console 
# first run prints the bootstrap password to the container logs
open https://localhost:8443/admin

# add your test domain to the allowlist, e.g. example.com
# configure one B1 policy with G2 (word assembly, 6 words)
III. Request a password as a user 
# public page — enter an email matching your allowlist
open https://localhost:8443/

# the captured email appears in /admin/captured-mail
# click the link, see N proposals generated in your browser
From
SealKeeper <noreply@your-company.example>
To
jdupont@your-company.example
Subject
Your SealKeeper password proposals are ready
Expires
15 minutes from issue
Single-use
Yes — link works only once

Hello,

Click the link below to reveal three password proposals tailored to your account level (B1). Pick the one you find easiest to remember. The proposals are generated entirely in your browser — SealKeeper's server never sees them.

seal.your-company.example/reveal/9f2a1b

If the link has expired or already been used, request a fresh one from the public page.

Security

Responsible disclosure

SealKeeper is a security product. We treat reports the way we would wish ours to be treated: promptly, in writing, without legal threat. Encrypted mail is preferred; plain text is accepted. We acknowledge within seventy-two hours and aim to publish a fix and advisory within thirty days of confirmation.

An informal bug-bounty programme rewards meaningful findings with a public acknowledgement, a printed certificate of thanks, and — at our discretion — a contribution to a security non-profit of the researcher's choosing.
