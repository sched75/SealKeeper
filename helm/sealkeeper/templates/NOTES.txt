3. First-time setup notes:

{{- if eq .Values.config.mode "eval" }}
   ⚠️  Evaluation mode is enabled. SealKeeper will print a bootstrap admin
       password to stdout on first boot. Check the pod logs:

         kubectl --namespace {{ .Release.Namespace }} logs deploy/{{ include "sealkeeper.fullname" . }} | grep "bootstrap admin"

       SQLite storage is local to the pod; do NOT use this mode in production.
{{- else }}
   ✅ Production mode. SealKeeper will refuse to start unless
       SK_MASTER_SECRET and SK_DATABASE_URL are populated. See:

         kubectl --namespace {{ .Release.Namespace }} describe deploy/{{ include "sealkeeper.fullname" . }}
{{- end }}

{{- if not .Values.persistence.enabled }}

   ℹ️  Persistence is disabled — uploaded libraries live in an emptyDir and
       will not survive a pod restart. Set `persistence.enabled: true` in
       values for a real PVC.
{{- end }}

{{- if not .Values.podDisruptionBudget.enabled }}

   ℹ️  PodDisruptionBudget is disabled — node drains can interrupt service.
{{- end }}
